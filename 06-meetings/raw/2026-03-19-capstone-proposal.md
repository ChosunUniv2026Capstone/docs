# 원문

## 프로젝트 주제

- 본 프로젝트는 조선대학교 기존 사이버캠퍼스를 개선한 차세대 사이버캠퍼스 프로토타입을 제작하는 것을 주제로 한다.
- 기존 사이버캠퍼스는 온라인 강의, 과제 제출, 시험, 출석 관리 등 핵심 학습 기능을 제공하고 있으나 출석 인증의 신뢰성, 기능 확장성, 유지보수 편의성 측면에서 개선 필요성이 존재한다.
- 본 프로젝트에서는 기존 LMS 핵심 기능을 현대적인 웹 서비스 구조로 재구성하고 Wi-Fi 기반 위치 판별과 디바이스 인증을 결합한 재실성 기반 출석체크 시스템을 추가한다.
- 본 과제는 웹 서비스 개발, 데이터베이스 설계, 네트워크 환경 구성, 사용자 인증, 시스템 보안, UI/UX 개선 등 전공 핵심 내용을 종합적으로 적용할 수 있다.

## 목적 및 필요성

### 배경

- 비대면·혼합형 수업 환경의 확대로 사이버캠퍼스 활용 비중이 증가하고 있다.
- 단순 로그인 또는 버튼 입력 중심 출석 방식은 대리출석, 비정상 출석 시도, 실제 재실 여부 확인에 한계가 있다.
- 시험, 출석, 온라인 강의 이수와 같은 학사 관리 기능은 공정성과 신뢰성이 중요하다.

### 목적

- 교내 Wi-Fi 환경과 연동해 학생의 실제 강의실 인접 여부를 반영하는 재실성 기반 출석 시스템을 마련한다.
- 기존 사이버캠퍼스 핵심 기능을 최신 프레임워크 기반으로 재구성해 사용성, 확장성, 유지보수성을 향상한다.
- 출석과 시험 기능에 인증 절차를 강화해 공정성과 신뢰성을 높인다.

### 개발 내용

- 기존 사이버캠퍼스 핵심 기능 재구현
- 공지사항, 강의자료, 온라인 강의, 과제 제출, 시험, 출석 기능 구현
- 최신 웹 프레임워크를 활용한 UI/UX 및 시스템 구조 개선
- Wi-Fi 기반 위치 판별 구조 적용
- AP 모드 공유기와 게이트웨이를 활용한 연결 단말 정보 수집
- 공유기별 연결 단말 정보와 사용자별 등록 단말 정보 매칭
- 사용자별 현재 연결 Wi-Fi 정보와 강의실 정보 연계
- 강의실 인근으로 판별된 사용자만 출석 가능하도록 로직 구현
- 사용자별 등록 단말 기반 접속 기기 식별
- 미등록 단말, 중복 단말, 비정상 접속 시도에 대한 판별 로직 반영

### 필요성

- 출석 및 시험 인증의 신뢰성을 높여 공정성 있는 평가 기반을 마련할 수 있다.
- 단순 출석 확인 절차를 개선해 수업 운영 효율성과 시스템 신뢰성을 높일 수 있다.
- 향후 교내 LMS 고도화 및 스마트캠퍼스 환경 구축을 위한 확장 가능한 모델로 활용할 수 있다.

## 목표 및 수행 내용

### 프로젝트 목표

- 차세대 사이버캠퍼스 프로토타입 구현
- Wi-Fi 기반 위치 판별과 디바이스 인증을 결합한 재실성 기반 출석 인증 체계 구현
- 출석과 시험 기능의 신뢰성 향상
- 웹 서비스 개발, 네트워크 연동, 인증 로직 설계, 데이터 모델링 역량 강화

### 수행 내용

- 학생, 교수, 관리자 관점 핵심 기능 도출
- 출석의 신뢰성, 학습 기능의 편의성, 관리 효율성을 반영한 요구사항 정리
- 웹·모바일 환경을 고려한 서비스 구성 방향 수립
- 강의실 Wi-Fi 환경과 연계한 출석 인증 구조 설계
- 강의실별 Wi-Fi 정보와 사용자 정보를 연계한 위치 기반 출석 판단 기능 구현
- 강의 시간, 사용자 정보, 강의실 정보를 종합한 출석 가능 여부 판단 로직 적용
- 사용자별 등록 단말 기반 접속 기기 식별
- 미등록 단말, 중복 단말, 비정상 접속 시도 탐지 설계
- 공지사항, 강의자료, 온라인 강의, 과제 제출, 시험 기능 재구현
- 재실 여부 판단이 필요한 기능에 접근 제어 강화

## 프로젝트 수행 방법

- 강의, 교수, 학생, 강의실, 강의실별 Wi-Fi, 수업시간 간의 관계 정의
- 출석 판정과 학습 기능 연동을 위한 데이터 구조 설계
- 사용자, 강의실, 단말, 네트워크 정보를 통합 관리할 수 있는 데이터베이스 모델링
- 강의실을 가정한 공유기(AP) 환경 구성
- 게이트웨이를 통한 접속 단말 정보와 사용자 정보 연계 환경 마련
- 학생의 현재 Wi-Fi 정보와 공유기 측 단말 연결 정보 수집
- 수집된 네트워크 정보와 사용자 등록 단말 정보를 비교하여 재실 후보 판정
- 학생의 출석 요청 시 수업 시간표, 강의실 정보, 네트워크 판정 결과를 종합하여 출석 허용 여부 결정

## 기대효과 및 활용 방안

- 대리출석 및 부정행위 방지
- 수업 운영 효율 향상
- 시험 공정성 강화
- 시스템 신뢰성 및 확장성 확보
- 전공 실무 역량 강화
- 캡스톤디자인 결과물, 전시, 경진대회, 후속 개발 자료로 활용
